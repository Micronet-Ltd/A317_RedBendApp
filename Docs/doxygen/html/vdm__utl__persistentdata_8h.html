Typedef Documentation

typedef void(* VDM_UTL_PersistentData_compatCodeCB_t)(const char *inOldVersion, const char *inCurrentVersion, E_VDM_PERSISTENT_UTL_ItemType_t inType)

Callback invoked to convert the old persistent storage items to the new format, as required. Each will be invoked only once, the first time that the new version of OMA DM Protocol Engine is run. Each callback is associated with a specific type of data.

Parameters
inOldVersionThe old OMA DM Protocol Engine version
inCurrentVersionThe new OMA DM Protocol Engine version
inTypeThe data type, an E_VDM_PERSISTENT_UTL_ItemType_t value

Enumeration Type Documentation

enum E_VDM_PERSISTENT_UTL_ItemType_t

Persistent storage data types. Used when invoking the compatability callbacks.

When you invoke the callbacks, you invoke all callbacks registered for a specific type of data. For example, all callbacks to update the DM Tree.

Enumerator:
E_VDM_PERSISTENT_UTL_DMTree 

DM Tree

E_VDM_PERSISTENT_UTL_Registry 

Configuration file (not used by vDM)

E_VDM_PERSISTENT_UTL_Last 

The type of data that will be updated last. After this type of data is updated, the storage is flashed.
