IBOOL VDM_Client_PL_logInit ( VDM_Handle_t  inLoggerContext)

Initialize platform-specific logging data and store logger context.

This API sets the logging context, which can be stored in a global pointer.

The logging component is initialized by the Engine once when it starts up.

Note
The OMA DM Protocol Engine API includes a context utility to allow the Framework components to manage their context. When implementing the logging component, do not use the context utility to store the logger context, as this may cause undefined behavior.
Parameters
inLoggerContextHandle to the logger component context that should be available for retrieval by the logger
Returns
TRUE on success, FALSE otherwise
void VDM_Client_PL_logMsg ( const char *  inFormat,
  ... 
)

Log input message.

Logging is done in two steps:

  1. The OMA DM Protocol Engine Logger utility calls VDM_Client_PL_logPrefix. This allows the logging component to add decorations to the text, such as color according to the severity level, a timestamp, and so on. Typical information also includes the thread id, file, line number, and component id from which the message is called.
  1. OMA DM Protocol Engine calls this function. This function writes the actual log message to the log console and/or log file.

Both functions take arguments similar to printf: a format string followed by a number of data arguments that get replaced within the string. For more information on printf format flags, see the printf manual page.

Note
This function is always preceded by a call to VDM_Client_PL_logPrefix. No extra decorations should be added to the input message.

Log prefix to an upcoming log message.

Logging is done in two steps:

  1. The OMA DM Protocol Engine Logger utility calls this API. This allows the logging component to add decorations to the text, such as color according to the severity level, a timestamp, and so on. Typical information also includes the thread id, file, line number, and component id from which the message is called.
  1. OMA DM Protocol Engine calls VDM_Client_PL_logMsg. This function writes the actual log message to the log console and/or log file.

Both functions take arguments similar to printf: a format string followed by a number of data arguments that get replaced within the string. For more information on printf format flags, see the printf manual page.

Note
This function is always followed by a call to VDM_Client_PL_logMsg.
